# Step 4: Module 2 — Severity Categorization Matrix

## Overview
This module acts as the logical bridge between the raw computer vision detections and the downstream escalation workflows. Its sole responsibility is to evaluate an incoming detection event and apply a risk severity tier (LOW, MEDIUM, HIGH, CRITICAL).

## Implementation Details
The `SeverityMatrix` (`src/severity/matrix.py`) is designed as an independent functional block to maintain modularity:
1. **Rule Initialization**: Upon instantiation, it loads the immutable `data/policy_rules.json` generated by the NLP Policy Parser in Phase 2.
2. **Context Evaluation**: When `assign_severity(detection_event)` is called, the matrix reads the `behavior_class` from the raw event payload.
3. **Classification Injection**: It queries the loaded policy rules to determine the hazard context defined in the original `Compliance_Policy_Manual.txt`. It then injects the corresponding `severity` field into the payload before returning it to the pipeline.

## Output
By separating this logic from the Detection Engine, we ensure that the computer vision model doesn't need to inherently understand risk analysis; it only detects behaviors. The matrix takes the record:
```json
{
  "behavior_class": "Carrying Overload with Forklift",
  ...
}
```
And transforms it by injecting the policy-derived severity context:
```json
{
  "behavior_class": "Carrying Overload with Forklift",
  "severity": "CRITICAL",
  ...
}
```
